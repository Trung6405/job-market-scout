# Work summary — 2026-07-23

Review-driven hardening of the Advisor stage and its persistence path. A
code review surfaced three weaknesses — false gaps from naive skill
matching, a partial-persistence window when a run dies mid-Advisor, and
several implicit contracts. Spec'd and planned the fixes under
`pipeline-hardening`, then began execution.

## 2. Phase 1 — Gap-matching accuracy

Stopped false gaps: a skill the student holds is no longer flagged missing
because a listing phrased it as a common variant.

| Commit | Summary |
|---|---|
| `feat(advisor): add normalize_skill() for gap matching` | Deterministic skill-name normalizer (version-suffix strip + alias fold) |
| `fix(advisor): match gaps on normalized skill names` | `evaluate_requirements` compares normalized skills both sides |
| `feat(advisor): ask extraction LLM for canonical skill names` | Extraction prompt requests single canonical names |

Phase 1 complete: 15 tests green across `test_advisor_gaps.py` and
`test_prompts.py`.

## 3. Phase 2 — Persistence robustness

Closing the partial-persistence window and making silently-dropped
listings visible.

| Commit | Summary |
|---|---|
| `fix(pipeline): warn when extraction drops scored listings` | Pipeline emits a warning event when the extraction LLM returns fewer listings than were scored |
| `fix(pipeline): persist run atomically in a single transaction` | `record_run_listings`→`finish_run`→renders wrapped in one transaction; scores no longer commit before the gap LLM pass |
| `test(pipeline): lock same-date re-run idempotency` | Test proving two same-date runs collapse to one row with upserted listings |

DB-backed atomicity test (`..._rolls_back_on_mid_persist_failure`) skips
locally (no Postgres in sandbox), verified via CI; non-DB agent ordering
tests green locally.
