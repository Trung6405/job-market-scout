## 6. Gap-kind-classification spec

Traced a second class of false gap, distinct from the naming-variant one
fixed in `pipeline-hardening` phase 1: non-skill requirements (degrees,
experience thresholds, soft skills) are string-matched against the tech
stack, can never match, and so always surface as unmet gaps — e.g. a "STEM
degree in CS" requirement flagged missing for a profile holding a CS
degree. Brainstormed and spec'd the fix: tag each extracted requirement
with a closed-vocabulary `kind` and only gap-match the `skill` kind;
non-skill requirements render as display-only context.

| Path | Summary |
|---|---|
